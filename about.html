About BashSnippets

This page is the straight answer to who runs BashSnippets.xyz, why I bothered to publish it, and how the pages actually get online. No agency spin—just what I do and how I work.

Who's Behind This

I’m Anguishe. I live in the Florida Panhandle and I’m self-taught on Linux—the kind of self-taught where nobody handed me a curriculum and I still stub my toe on edge cases weekly.

Before any of this, I worked beach hospitality. Then I had spinal fusion surgery and ended up stuck at home for a long recovery window. With too much time and a machine in front of me, I wiped the Windows installs off everything I owned and put Kali Linux on instead—mostly as a sink-or-swim move. If I was going to sit still, I wasn’t going to spend it in the shallow end.

I learned how I learn best: YouTube when someone explains a concept cleanly, AI tools when I’m stuck on syntax, and a lot of breaking stuff until it behaves. I’m about two years deep now. Some days I feel sharp; other days I’m rereading man pages for things I thought I already knew. That’s normal, and I leave it in because nobody improves by pretending they shipped fully formed.

Why This Site Exists

I started BashSnippets because I needed to get serious about cron automation and because I came from zsh and kept tripping over small differences when I jumped into bash. The internet has answers, but I was hitting the same searches over and over—half-remembered flags, syntax that looks identical until it isn’t, and one-liners I knew I’d solved before.

So I turned my notes into something public: a reference I’d actually trust because I wrote it while debugging real tasks. Over time it stopped being “just for me” and turned into a library I’m fine sharing. Every script here is either in my rotation now or was written for something I actually needed at least once. If it’s fluff, it doesn’t stay.

What I Actually Work With

At home I run a server stack on Proxmox with Tailscale layered on top. That whole setup is on pause right now while I sort hardware upgrades—same lab rat habits, just waiting on parts instead of configs.

I’m juggling a few web projects. One you might bump into is kaisrun.xyz. BashSnippets itself is hand-coded static HTML on GitHub Pages: no CMS, no frameworks, no build pipeline to babysit. On the side I’m also tinkering with a small music app for my daughter—mostly vibe-coded, mostly for fun, and a good excuse to practice shipping UI without overthinking it.

How This Site Is Built

The stack is boring on purpose: HTML, CSS, and vanilla JavaScript. The site lives on GitHub Pages. Each page is typed out by hand instead of poured out of a template engine, which is slower but keeps me honest about what’s actually on the wire.

I don’t run WordPress or a headless CMS here—the repo is the site. Scripts in the library get exercised on Ubuntu 22.04 LTS before I call them done. The interactive tools run real logic in the browser; they’re not hollow “generator” shells that paste canned text. If something claims to build a snippet, it’s doing the work.
